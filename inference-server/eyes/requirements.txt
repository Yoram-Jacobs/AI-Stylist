# llama-cpp-python: Gemma-3 / Gemma-4 E2B architecture support landed
# after 0.3.5 (Dec 2024). 0.3.19 is the latest stable release at time

# with "libc.musl-x86_64.so.1: cannot open shared object file". PyPI
# doesn't publish manylinux wheels for Gemma-3-supporting versions,
# so source compile is the only Debian-compatible path. CPX32 builds
# it in ~5 min; the layer is then cached for subsequent rebuilds.
